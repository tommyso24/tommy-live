给 AI 装上超能力，然后它帮我造了个评论系统

发布于 2026-04-04 · 学习记录

今天早上研究了一个叫 Superpowers 的开源技能框架——装上之后，AI 编程助手不再是上来就甩代码，而是先跟你聊需求、画架构、写计划，确认了再动手。

为了测试效果，我让它帮我给博客做了一套完整的评论系统。从数据库设计到前端组件到管理后台，全套。

Superpowers 是什么

一句话：它是一套开源技能框架，让 AI 编程工具自动遵循专业软件工程流程。

平时用 Claude Code 或者 Cursor，说「帮我加个功能」，AI 通常上来就写代码。写出来能不能用？看运气。

装了 Superpowers 之后，AI 的行为模式完全变了：

  1. 先问你需求 — 不急着写代码，先通过提问帮你理清到底要什么
  2. 再出设计方案 — 画架构图、选技术栈、列出文件结构
  3. 然后制定计划 — 把任务拆成 2-5 分钟的小步骤，每步写什么、测什么、怎么验证
  4. 你确认了才动手 — 说「go」之后，它按计划一步步执行

简单说就是从「随便写写」变成了「需求分析 → 设计 → 计划 → 执行」的正经流程。GitHub 130k+ Stars，MIT 开源，完全免费。

怎么装

在 Claude Code 里一行命令：

/plugin install superpowers@claude-plugins-official

装完重启 Claude Code 就生效了。不需要记任何命令，用自然语言说「帮我做个 XX 功能」就能触发整套流程。

怎么确认装好了？跟 AI 说「我想给这个项目加个新功能」——如果它开始问你需求细节而不是直接写代码，就说明在工作了。

实战：用 Superpowers 从零搭评论系统

光说不练假把式。我直接拿我这个博客开刀——之前文章没有评论功能，读者看完了没地方说话。

阶段一：头脑风暴（AI 问，我答）

我跟 AI 说：「我想给博客文章加评论功能」。

装了 Superpowers 之后，它没有直接开始写代码，而是问了我一串问题：

  • 需要登录才能评论吗？ → 不要，匿名的，填个昵称就行
  • 要不要楼中楼回复？ → 要，但最多两层就够了
  • 评论需要审核吗？ → 要，我手动审核通过才显示
  • 管理后台怎么进？ → 密码保护的独立页面
  • 技术栈有偏好吗？ → 用 Cloudflare Pages Functions + D1，跟现有部署一致
  • 风格？ → 跟站点像素风统一

这些问题我之前自己都没全想清楚。被 AI 问一遍之后，需求反而更明确了。

阶段二：设计文档

问完之后，AI 生成了一份完整的设计文档，包括：

  • 架构图（浏览器 → Pages Functions → D1 数据库）
  • 数据库表结构（comments 表，带 slug、nickname、content、status、parent_id）
  • API 设计（公开的 GET/POST + 管理员的 GET/PATCH/DELETE）
  • 安全措施（IP 哈希限频、XSS 防护、参数校验）

我看了一遍，调了几个细节（比如限频改成 60 秒一条），确认通过。

阶段三：实施计划

然后 AI 把整个项目拆成了 7 个任务：

任务内容
Task 1数据库 Schema + Wrangler 配置
Task 2公开评论 API（GET 获取 + POST 提交）
Task 3管理员 API（审核/删除）
Task 4评论组件样式（像素风 CSS）
Task 5评论组件逻辑（vanilla JS）
Task 6嵌入所有博客文章页面
Task 7管理后台页面

每个任务都标了要改哪些文件、写什么代码、先测什么。我确认后说「go」，它就开始一个一个执行了。

阶段四：自动执行

接下来就是看它干活了。每完成一个任务会自动 commit，我可以随时看进度。

最终产出：

  • 后端 API — 两个 Cloudflare Pages Functions，处理评论的增删查改
  • D1 数据库 — 一张 comments 表，带索引，支持楼中楼
  • 前端组件 — 纯 vanilla JS + CSS，像素风，支持回复和字数统计
  • 管理后台 — 密码保护，可以审核/删除评论，带筛选和搜索
  • 全站集成 — 所有博客文章底部自动嵌入评论区

整个过程不到两小时。如果没有 Superpowers 的流程约束，AI 大概率会跳过设计直接写代码，然后中途发现架构不对再推翻重来。

有 vs 没有 Superpowers

对比没装装了
开局AI 直接写代码AI 先问需求细节
设计没有生成设计文档，你确认后才开始
计划没有拆成小任务，标清文件和步骤
执行一口气输出一大堆按计划逐步执行，每步可验证
出错时不知道哪步错了精确到具体任务，回退成本低

几个实用技巧

  1. 认真对待头脑风暴。 这是 ROI 最高的环节，花 10 分钟能省后面 2 小时。
  2. 一次只做一个功能。 范围太大会让计划变得混乱，保持聚焦最有效。
  3. 不需要记命令。 用自然语言就行，「帮我规划这个功能」就能触发全流程。
  4. Session 崩了也不怕。 计划文档里的 checkbox 记录了进度，新会话从断点恢复。
  5. 计划阶段就提修改。 不要等到执行时才发现不对，那时候改成本高。

最后

Superpowers 最大的价值不是让 AI 写更多代码，而是让它在写代码之前先想清楚。

这跟我之前写的 SaaS 原型那篇文章说的是同一个道理——AI 是执行力最强的搭档，但它需要方向。Superpowers 做的事情，就是给这个方向加了一套流程保障。

对了，你现在就可以试试这篇文章底部的评论区——那就是用 Superpowers 搭出来的。
