一个下午 + 一个晚上，AI 帮我搭了整个 SaaS 平台的前端原型

发布于 2026-04-04 · 碎碎念

对，你没看错。一个下午加一个晚上。15 个页面。从营销首页到后台管理系统。全部可交互。

而我做的最多的事情，是「不对不对，改一下」。

起因：一个外贸人的野心

事情是这样的。我们公司有一个旧的外贸建站系统叫「超新星」，跑了几年了，客户还行。但我一直觉得光建站不够——外贸人的痛点是全链路的，建站只是第一步，后面还有询盘管理、买家背调、跟进策略……

所以我想做一个平台，把三个产品整合到一起：

  1. 超新星建站 — 老本行，高颜值外贸独立站
  2. MML表单 — 自研 WordPress 询盘插件，带 AI 背调和国别过滤
  3. 秒探客 — 60 秒 AI 买家情报报告

听起来工程量很大对吧？如果用传统方式找设计师出图、前端写页面，没个月也拿不下。

但是 2026 年了，我有 Claude。

第一阶段：从 0 到 1（下午 2 点 - 6 点）

先聊天，一行代码不写

我没有一上来就让 AI 写代码。而是先跟它聊了半小时——把整个产品的定位、三个产品的关系、注册漏斗的逻辑、后台的导航方式全部讲了一遍。

AI 很快就理解了，还给我画了一个 ASCII 架构图（对，就是用文本符号画的那种）。看了一下，大方向没问题，但有几个细节要调：

  • 后台导航要用「顶部 Tab 切换产品 + 左侧子菜单」的两层结构
  • 公司资料库要作为独立顶级入口，不藏在设置里
  • 每个产品的后台是独立页面，不搞 SPA

这些决策一旦确认，后面就快了。

文件夹重组 + 首页

我把之前做的一个秒探客注册流程页面丢给 AI 看了看样式，说「照这个风格做」。然后它：

  1. 把文件夹重新组织了一遍（tools/、onboarding/、app/ 分层）
  2. 去爬了我们旧站 www.msupernova.com 的内容
  3. 用新的 Apple 质感风格重新设计了首页

全程我只需要给意见。比如我说「Hero 的副标题要按业务流来写——建站、表单、背调」，它就给了我几个方案选，我选了一个，改了几个字，搞定。

MML表单产品页

这个页面花了比较多时间跟 AI 讨论文案。比如「AI 自适应表单样式设计」这个卖点，我觉得太技术向了，我们俩头脑风暴了几轮：

AI: "零 CSS 基础，一键适配网站风格"
我: 太长了，要跟其他 bullet point 长度一样
AI: "一键匹配网站风格" / "零基础 AI 设计表单" / "截图即设计，零代码"
我: 零代码，一键适配网站风格

最后定稿的这句，其实是我从 AI 给的几个选项里拼凑改了几个字拼的。AI 是很好的文案引擎，但最后的决策还是得人来拍板。

第二阶段：后台系统（晚上 7 点 - 11 点）

这部分是最有成就感的。一个晚上做了：

模块包含内容
后台框架顶栏 Tab + 侧栏 + 内容区
充值页微信/支付宝扫码 + Token 套餐 + 开票 + 对公转账
账单页4 个 Tab（全部/充值/购买/Token 消耗）
建站系统面板控制面板/页面管理/模板库/询盘管理/数据分析
MML表单面板概览/询盘列表/购买定价/License/下载/推送设置/插件设置
公司资料库公司信息/产品线管理/文件资料
秒探客面板数据概览/新建秒探/历史报告/偏好设置
个人中心个人信息/账户安全/我的订阅

每个页面我都会本地预览、截图看效果、提修改意见。AI 改起来也快，基本上我说一句，它几秒钟就改好了。

一些有趣的插曲

Vercel 部署被 block 了。 因为我本地 git 的提交者是 苏鹏 <supeng@Tommys-MacBook.local>，Vercel Hobby 计划不认识这个用户，以为是「团队协作」结果 block 了。解决方法：把 git 的邮箱改成 GitHub 注册邮箱，force push 一下。白白浪费了 20 分钟排查。

dashboard.html 被我自己弄丢了。 之前做了一个 dashboard.html 作为后台框架（里面有秒探客的完整内容），后来我让 AI 把它改名为 mml-leads.html，结果秒探客的内容就没了。最后从 git 历史里找了回来。教训：git log 是你最好的后悔药。

侧栏出了 bug。 AI 在合并代码的时候，把「购买」和「License」两个菜单项重复写了两遍。我截了个图扔过去，它秒速修好。这就是为什么一定要一边做一边预览——不要等全做完再看，那时候 bug 已经叠了一层了。

我的工作流总结

我讲清需求和背景（别省）
↓
AI 出架构方案，我确认关键决策
↓
AI 写页面，我本地预览
↓
我截图 + 标注反馈
↓
AI 修改，我验收
↓
满意 → push 到 GitHub → Vercel 自动部署
↓
重复，直到所有页面完成

整个过程中我没有写过一行 CSS。但我做了所有的设计决策：配色、布局、文案、交互逻辑、信息架构。

AI 是执行力最强的搭档，但它不是产品经理。它不知道「建站→表单→背调」这个业务流的顺序哪个字母是最优摆放最重要。它不知道 MML表单的表单管理和 AI 设计功能应该在 WordPress 后台里，不应该放在我们的 SaaS 后台。这些判断，只有懂业务的人能做。

最终成果

类型页面数包含内容
营销前台3首页、秒探客产品页、MML表单产品页
注册漏斗3秒探客 2 步 + MML表单 1 步
用户后台7建站/秒探客/MML表单/公司资料库/充值/账单/个人中心
辅助页面2sitemap + CHANGELOG
总计15全部可交互，含完整导航

部署在 Vercel 上，GitHub push 自动部署。从第一行代码到全部上线，大概 8 小时。

给想用 AI 做项目的人的建议

  1. 先聊天再动手。 花 30 分钟把需求讲清楚，能省后面 3 小时的返工。
  2. 分阶段确认。 不要一口气让 AI 做完所有页面再看。做一个看一个，及时纠偏。
  3. 截图比文字好使。 说「左边导航栏不对」不如直接截图画个箭头。
  4. 文案要自己拍板。 AI 给的文案 80 分没问题，但从 80 到 95 需要你自己的判断。
  5. git 是你的安全网。 大胆让 AI 改，改坏了从历史版本找。
  6. 不要怕推翻重来。 我中间把整个文件夹结构重组了两次，把 dashboard 改名又改回来，都没关系。纯 HTML 的好处就是成本低。

最后

有人可能会说，「这不就是原型吗，又不是真正的产品。」

没错，但原型的意义就在于——它让所有人（包括我自己）第一次能看到这个平台长什么样。 在此之前，这些东西只存在于我的脑子里和几页 PPT 上。

现在，我可以把链接甩给同事说，「点开看，这就是我们要做的东西。」

然后我们就可以开始真正的开发了。

而这一切，只花了一个下午加一个晚上。
