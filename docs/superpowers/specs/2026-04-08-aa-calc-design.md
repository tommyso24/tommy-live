# AA 记账神器 — 设计规格

## 概述
纯前端聚餐分账工具，3 步引导式操作，支持均分和指定人两种分摊方式，用完即走不需要账户。

## 操作流程

### 第 1 步：添加参与人
- 输入框 + "添加"按钮，逐个添加参与人名字
- 名字长度限制 1-10 字符
- 至少 2 人才能进入下一步
- 已添加的人显示为标签，可点 × 删除
- 点"下一步"进入费用录入

### 第 2 步：添加费用项
- 每项费用包含 3 个字段：
  - **名称**（如"锅底"、"烤鱼"）— 1-20 字符
  - **金额**（数字，精确到分）— 0.01-99999
  - **分摊方式**：
    - 均分：所有人平摊
    - 指定人：显示参与人复选框，勾选参与的人
- 添加按钮新增一项费用
- 已添加的费用显示为列表，可删除
- 实时显示费用总计
- 至少 1 项费用才能进入下一步
- 可点"上一步"回去改人员
- 点"算账"进入结果页

### 第 3 步：查看结果
- **每人应付明细**：列出每个人的应付金额（分项 + 合计）
- **垫付人选择**：下拉框选"谁先付了全部钱"
- **转账指引**：选择垫付人后，显示"A → 垫付人：¥XX"的简洁转账列表
- **重新开始**按钮：回到第 1 步
- 排版清晰整齐，适合截图发群

## 计算逻辑

### 每人应付
- 均分项：金额 / 总人数（四舍五入到分）
- 指定人项：金额 / 被指定的人数（四舍五入到分）
- 尾差处理：分摊后如有 1-2 分差额，加到第一个人身上

### 转账方案
- 选定垫付人后：每人应付 - 0 = 应转账金额（垫付人自己的部分不用转）
- 垫付人应收 = 总金额 - 垫付人自己应付的金额
- 显示格式：`张三 → 李四（垫付人）：¥52.50`

## UI 设计

- 配色沿用 CSS 变量
- 星星背景 + 扫描线

## 技术约束
- 纯 HTML / CSS / JavaScript，无框架
- 纯前端计算，无 API 调用，无数据持久化
- 复用：`site-chrome.js`、`site-chrome.css`
- 不需要 `auth.js` 和 `avatars.js`（无账户功能）
- 响应式设计，手机端友好
