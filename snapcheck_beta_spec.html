01战略与目标

为什么 Beta 期不卖套餐、不直接定价？

核心战略：用数据定价，不用拍脑袋

我们之前基于 Aihubmix 做过一份 ¥88/¥228/¥588 的定价方案。但现在切换到 Liaobots 后成本下降了 70%+，而且用户的真实使用习惯（联网 vs 不联网比例、月均次数、功能偏好）我们其实不知道。

所以这次 Beta 期的设计是：15 天内不公布套餐价格，只开放免费试用 + 按次加购两种入口。15 天后我们会拿到三组关键数据，再据此决定正式套餐：

  • 注册转化数据：明天活动 + 14 天累计能拉到多少注册
  • 真实使用数据：联网/不联网的实际比例、平均使用率
  • 付费意愿数据：加购转化率、加购金额、不联网/联网加购偏好
Beta 期的第一目标

不是赚钱，不是验证套餐价格，是让尽可能多的人注册账号 + 收集真实使用数据。所有产品和运营动作都为这个目标服务。

三个不做

  • 不做套餐：Beta 期完全不出现"月付/年付"概念，避免干扰用户行为
  • 不做套包：加购只有"不联网 ¥0.5/次"和"联网 ¥1.5/次"两个 SKU，不搞买多送多，让加购数据成为最纯净的偏好信号
  • 不做 CRM 对接：永远被动，让客户的 CRM 来对接我们

免费试用 + 按次加购，简单到极致

加购 SKU（极简两选项）

这是我们 Beta 期收集"用户偏好信号"的核心机制。不做套包、不做买多送多、不做阶梯折扣——保持极致简单，让每次加购都成为一次明确的偏好投票。

✓ 为什么这样设计

用户每次加购就是一次明确投票——他选了 30 次不联网而不是 30 次联网，就是明确告诉我们"这个用户更倾向不联网场景"。15 天结束我们能拿到一份非常干净的"用户偏好报告"，定价时再不用猜。

加购页面用户体验

用户免费额度用完后的引导页要做温和提示而不是强推加购，给两个选择：

  • 立即加购 → 跳转到加购页（强付费意愿信号）
  • 通知我正式版 → 留邮箱即可（弱意愿但仍是数据）

加购页底部必须显示一行提示："正式版发布后均价将更优惠，您作为创始会员将自动享有最低折扣。" — 这是 Day 15 转化的伏笔，不是阻拦加购。

09Liaobots 接入

这是明天上线最关键的依赖，必须先打通

前置准备

  1. 注册账号：去 liaobots.com（或他们的中转 API 站点）注册
  2. 充值：建议先充 ¥500–1000 作为 Beta 期 buffer
  3. 获取 API Key：在控制台生成一个 Key，妥善保管
  4. 确认可用模型：登录后查看可用模型列表，确认 gemini-3-pro（或类似名称）可用
  5. 确认计费模式：和官方确认是按 token 计费还是按次计费，记录每个模型的实际单价
⚠️ 一定要确认的事

Liaobots 是中转代理，API 端点和参数命名可能不完全等同 Google 官方。明天上线前必须实测一次完整调用，把"模型名 / 端点 URL / 鉴权方式 / 返回结构"全部跑通，不要假设。

不联网背调的完整流程

  1. 用户点击"开始背调"，前端 POST /api/check/basic
  2. 后端校验额度 → 锁定 1 次（事务开始）
  3. 构造 prompt（system + user，详见下方 prompt 模板）
  4. 调用 Liaobots gemini-3-pro，max_tokens=4000
  5. 解析返回的 content，提取结构化字段
  6. 记录 token 数 → 计算成本写入 background_checks 表
  7. 事务提交，返回结果给前端

联网背调的完整流程（核心控制点）

  1. 用户提交公司名 → POST /api/check/web
  2. 校验联网额度 → 锁定 1 次
  3. 第一步：调用智谱搜索 API，搜索 "{company_name} {country}"
    • 智谱搜索的具体接口需要技术同事去查智谱开放平台文档
    • 必须设置返回结果数 ≤ 5 条
    • 必须截断每条结果的内容到 800 token 以内
    • 整体搜索结果合计不超过 4000 token
  4. 第二步：把搜索结果作为 context 拼进 prompt，调用 Liaobots gemini-3-pro
  5. 记录两次调用的总成本（智谱 + Liaobots），目标控制在 ¥0.14 以内
  6. 解析、存储、返回
⚠️ 成本护栏（必须实现）

联网背调的单次成本目标是 ¥0.14。技术实现上必须做以下硬性约束：

  • 智谱搜索结果 token 总数 ≤ 4000（超出截断）
  • Liaobots 调用 max_tokens ≤ 4000
  • 每次调用后实时计算成本，超过 ¥0.30 的请求自动告警（飞书通知）
  • 单用户每分钟最多 5 次背调，防止脚本刷量

你是一个专业的外贸客户背调分析师...

你是一个专业的外贸客户背调分析师...

{search_results}

基于以上信息，为以下公司生成背调报告：
公司名称: {company_name}
... (字段同上，但 risk_score 需要结合最新信息调整)
